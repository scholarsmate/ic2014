Innovation Challenge 2014
DMarcate

 

Innovators:
 Davin Shearer
 Bryan Jacobs
The Problem: Network traffic volumes have grown far faster than analysts' ability to review it.
The Only Solution: Automate some of the work of analysis.

Retain everything (or as much as possible), review only "important" content

This is the Novetta Cyber Analytics Product

 

NCA fundamentally is a tool for helping analysts find the gold in a pile of straw.

 

This is primarily done in two ways:

  • By making both metadata and raw session contents searchable
  • By checking incoming traffic for interesting patterns
As time has gone on,

 

the sophistication and complexity of the pre-filtering a packet must go through before being reviewed by a human has increased

and increased

and increased...

What if we could go back to basics and get the same quality results?
DMarcate:
combines Novetta Cyber Analytics' high-speed packet processing with WPText's matching engine
attaches annotations (labels) to streaming content based on what it finds
produces annotations from content bodies, not just from structured metadata

 

Nothing is missed.
When combined with NCA’s high-speed packet dispatch, these annotations can be used to route flows to more expensive processing.
sorts the wheat from the chaff
Example:
DMarcate can determine if a stream contains email traffic and send it to a separate parser which separates the header information, extracts attachments, ...
Annotations may also be used for tag searches, for statistical analysis, or for alerting
DMarcate is
Fast

Has a faster matching engine than its peers

This lets it handle more data in the same time period

Or search the same amount of data for more targets

DMarcate can do real-time analysis on petascale traffic.

Simple

Delivers immediate value without configuration

DMarcate can be configured by a non-programmer.

 

At its heart lie simple lists of textual terms.

Cost Effective
The matching engine already exists
One easy-to-integrate library
Runtime efficiency means lower hardware costs
Flexible
Language agnostic (unlike natural language parsing solutions)
No issues with text-speak, with slang, or with fragmented or incomplete thoughts
Uses a small amount of memory

 

Similarly configured instances can share

 

Possible to have many different matching engines running on less hardware
Many possible purposes:
  • malware detection
  • sentiment analysis
  • watchword triggering
Handles semi-structured content
Has a powerful rules engine underneath, if the deployment requires it
DMarcate can learn
A “corrections” file can be dynamically applied as the system runs

 

Mistakes made in the past do not have to continue to be mistakes in the future
Rules updates can be as simple as editing a text file
A scoring system delivers the most relevant annotations in ambiguous cases
What’s different about DMarcate? Where’s the Innovation?
Back-to-basics approach

 

You don’t have to parse everything. K.I.S.S, worse is better.

 

Requires less hardware, requires fewer engineers.
Simple configuration

 

Doesn’t require an English or Statistics Ph.D.

 

Can be driven (and improved!) by a data scientist end user.
Versatility

 

Equally well suited for difficult tasks (malware signature detection) and easy ones (email address harvesting)
